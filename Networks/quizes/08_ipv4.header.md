What is the fixed binary value of the first field of an IPv4 header?

A) 0010
B) 0110
C) 0001
D) 0100

### Anwser
Anwser is D

### Explaination
The first field of an IPv4 header is the Version field, and it is 4 bits long.

4 (decimal) to (binary) is 0100

## Quiz 2
Which field will cause the packet to be dropped if it has a value of 0?

A) TTL
B) DSCP
C) IHL
D) ECN

### Anwser
anwser is A.

### Explaination
- **TTL (Time To Live)** is an 8‑bit hop counter set by the sender. Each router that forwards the packet **decrements TTL by 1**.  
- If TTL reaches **0**, the router **drops the packet** to prevent infinite routing loops and typically sends an **ICMP Time Exceeded** message back to the sender.  
- **Why the others are not correct:**
  - **DSCP** = 0 just means default/best‑effort service; it does **not** cause a drop.
  - **ECN** = 0 means not ECN‑capable; it does **not** cause a drop (routers may mark congestion bits, but zero itself is fine).
  - **IHL** = 0 would be an invalid header length and a malformed packet would be discarded, but by specification IHL has a **minimum valid value of 5**.

## Quiz 3
How are errors in an IPv4 packet's encapsulated data detected?

A) The IPv4 Header Checksum field checks for errors.
B) The encapsultated protocol (TCP, UDP) checks for errors.
C) Errors in the encapsulated data cannot be detected.

### Anwser
Anwser is B.

### Explanation
- The **IPv4 Header Checksum** only covers the IPv4 **header**, not the payload, so it cannot detect errors in the encapsulated data.  
- Error detection for the payload is performed by the **transport or encapsulated protocol**:
  - **TCP** uses a checksum that covers a pseudo‑header, the TCP header, and the TCP data.  
  - **UDP** has a checksum that covers a pseudo‑header, the UDP header, and the UDP data; in IPv4 the UDP checksum can be zero (optional) but is commonly used.  
  - **ICMP** and other protocols also include their own checksums.
- Therefore, payload errors are detected by the encapsulated protocol (answer B).

## Quiz 4
Which field of an IPv4 header is variable in Length?

A) Options
B) Header Checksum
C) Total length
D) IHL

### Anwser
anwser is A

### Explanation
- **Options** is the only header *field* whose **length in bytes can vary** (0–40 bytes).  
- When options are present the **IHL** (4‑bit field) increases to reflect the longer header (IHL counts 32‑bit words).  
- **Header Checksum** is always 16 bits (fixed).  
- **Total Length** is a 16‑bit field whose *value* varies with packet size, but the field itself has a fixed length of 16 bits.  
- **IHL** is a fixed 4‑bit field that indicates header length; it is not itself variable in bit size.

## Quiz 5
Which bit will be set to 1 on all IPv4 packet fragments except the last fragment?

A) fragment offset bit
B) more fragments bits
C) don't fragment bit
D) packet fragment bit

### Anwser 
anwser is B.

### Explanation
The MF (More Fragments) bit in the IPv4 Flags field tells whether more fragments are coming.
If MF = 1, the fragment is not the last one — more fragments follow.
If MF = 0, it is the final fragment, and no more pieces will arrive.
When a large packet is split into fragments, every fragment except the last one has MF = 1, so the receiver knows it must wait for more before reassembling the full packet.
